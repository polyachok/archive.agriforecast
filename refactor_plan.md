# План рефакторинга и история изменений

## Описание проекта

Проект представляет собой веб-портал "Agriforecast" для мониторинга данных с агрометеорологических устройств. 

### Технологический стек:
- **Бэкенд:** PHP
- **Фронтенд:** HTML, CSS, JavaScript (с использованием Spectre.css)
- **База данных:** MySQL/MariaDB (две базы: `portal_db` для основной логики и `forecast` для данных)

### Основные сущности:
- **Пользователи (Users):** С ролями (Администратор, Дилер, Пользователь).
- **Устройства (Devices):** Двух типов: `VP` (датчик влажности и температуры почвы) и `M` (метеостанция).
- **Организации (Organizations):** Пользователи принадлежат организациям.

## Цель рефакторинга

Основная цель - переработать страницу дашборда (`pages/dashboard.php`) для упрощения и добавления новой функциональности.

## План действий

1.  **Создать файл плана `refactor_plan.md`**: Готово.
2.  **Создать копию `pages/dashboard.php` в `pages/dashboard_new.php`**: Создать новый файл для безопасной работы.
3.  **Упростить `dashboard_new.php`**:
    - Удалить текущую логику отображения виджетов устройств.
    - Очистить связанный CSS и JavaScript.
    - Оставить основную структуру страницы (шапка, сайдбар).
4.  **Добавить выбор года**:
    - Создать функцию для получения списка уникальных годов из таблицы `portal_db.device_forecast_value`, столбец `year`.
    - Реализовать на странице `dashboard_new.php` выпадающий список (`<select>`) с этими годами.
    - При смене года страница должна перезагружаться с новым значением года в GET-параметре.
5.  **Интегрировать VP-виджеты с выбором года**:
    - Восстановить базовый внешний вид VP-виджетов.
    - Для каждого VP-виджета отображать его ID.
    - Внутри каждого VP-виджета разместить выпадающий список выбора года (используя `Device->getForecastYears()`).
    - При выборе года в выпадающем списке осуществлять перенаправление на страницу `/pages/device_forecast.php?device_id=<ID_ПРИБОРА>&year=<ВЫБРАННЫЙ_ГОД>`.
